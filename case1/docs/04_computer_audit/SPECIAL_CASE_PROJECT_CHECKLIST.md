# Special Case Project Checklist: 工程承攬專案稽核 Checklist 初版

文件狀態：初版  
Special Case：疑似違建 / 未合法申報施工  
定位：Computer Audit 專案稽核檢核表  
重要聲明：本 Checklist 用於發現風險與蒐集證據，不用於自動判定違建或取代稽核主管與專業人員判斷。

## 1. 使用方式

本 Checklist 以專案全生命週期為檢核單位。

建議使用順序：

1. 先確認專案基本資料與合法性文件。
2. 再比對合約、圖說、現場、估驗、請款與驗收。
3. 再檢查 ERP / Workflow / Audit Trail 是否支撐同一條證據鏈。
4. 最後由稽核主管或稽核人員判斷是否形成稽核發現、待釐清事項或改善建議。

## 15. Red Flags 總表

| Red Flag | 風險意義 | 建議稽核動作 |
| --- | --- | --- |
| ERP 專案已建立，但合法性文件欄位空白 | 系統流程可能未綁定合法性文件 | 要求補齊文件或提供免辦判斷 |
| Workflow 已核准，但附件版本不明 | 形式簽核不足以支撐證據鏈 | 查核附件版本與補傳紀錄 |
| 施工日早於變更核准日 | 可能先施工後補件 | 建立 Suspicious Timeline |
| 估驗包含未核准變更項目 | 未核准工程進入計價 | 比對估驗、合約、變更單 |
| 請款早於估驗核准 | 請款基礎不足 | 查核 AR / Billing 流程 |
| 現場照片與圖說不一致 | 可能超範圍或未核准施工 | 要求專業人員判讀與現場複核 |
| 保留款未驗收即釋放 | 結案控制失效 | 查核保留款釋放條件 |
| 同一人多重權限 | SoD 失效 | 產出權限衝突清單 |

## 16. 稽核結論欄位

| 欄位 | 說明 |
| --- | --- |
| 初步風險等級 | High / Medium / Low |
| 主要異常類型 | 文件不足 / 時序異常 / 圖說不一致 / ERP 落差 / Workflow 附件不足 |
| 待補證據 | 列出缺少的文件或系統資料 |
| 需訪談對象 | 業務 / 工程 / 財務 / 法務 / 資訊 / 監造 / 專案經理 |
| 稽核判斷 | 最終由稽核人員記錄，不由系統自動填寫 |
| 改善建議 | SOP / Workflow / ERP / 權限 / 文件治理改善 |
