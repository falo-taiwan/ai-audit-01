Special Case Project Checklist: 工程承攬專案稽核 Checklist 初版

文件狀態：初版 Special Case：疑似違建 / 未合法申報施工 定位：Computer Audit 專案稽核檢核表 重要聲明：本 Checklist 用於發現風險與蒐集證據，不用於自動判定違建或取代稽核主管與專業人員判斷。

1. 使用方式

本 Checklist 以專案全生命週期為檢核單位。

建議使用順序：

  1. 先確認專案基本資料與合法性文件。
  2. 再比對合約、圖說、現場、估驗、請款與驗收。
  3. 再檢查 ERP / Workflow / Audit Trail 是否支撐同一條證據鏈。
  4. 最後由稽核主管或稽核人員判斷是否形成稽核發現、待釐清事項或改善建議。

Red Flag風險意義建議稽核動作

Workflow 已核准，但附件版本不明形式簽核不足以支撐證據鏈查核附件版本與補傳紀錄
施工日早於變更核准日可能先施工後補件建立 Suspicious Timeline
