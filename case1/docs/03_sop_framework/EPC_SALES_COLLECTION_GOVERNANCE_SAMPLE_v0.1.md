# 工程承攬業銷售與收款循環作業程序 S版 v0.1

文件狀態：Sample Version / 市場現況 baseline  
文件定位：台灣上市櫃公司常見傳統 SOP / 內控制度寫法示範  
適用範圍：工程承攬業銷售與收款循環  
主要讀者：董事會、稽核室、財會、業務、工程、ERP 顧問與部門主管  

## 0. 文件聲明

本文件為傳統上市櫃治理 SOP 示範樣貌，用來呈現台灣企業目前常見的內控制度與作業程序文件寫法。

本文件不是簡化版，也不是教學版，更不是錯誤版本。它代表的是市場主流現況：許多公司目前就是以這種方式描述流程、權責、核准、表單與文件留存。

本文件：

- 為傳統上市櫃治理 SOP 示範樣貌。
- 非 TAAT Governance 進化版。
- 非 Computer Audit 深度版。
- 非 AI-assisted Governance 文件。

本文件用途：

- 作為「市場現況 baseline」。
- 供後續與 A 版治理進化版及 B 版 Computer Audit 深度版對照。
- 協助一般內稽、財會、ERP 顧問與主管辨識傳統 SOP 的既有樣貌。

## 1. 目的

為建立工程承攬業銷售與收款循環之標準作業程序，明確規範合約承接、合約變更、工程估驗、請款、收款、保留款、驗收及文件留存等作業，確保交易經適當核准、金額記錄正確、款項收取完整，並作為內部控制制度執行與稽核查核之依據。

## 2. 適用範圍

本程序適用於本公司工程承攬案件之銷售與收款循環，包含下列作業：

1. 客戶與案件基本資料建立。
2. 合約審查、簽訂與歸檔。
3. 合約變更及追加減作業。
4. 工程進度確認及估驗作業。
5. 請款、發票開立及應收帳款入帳。
6. 收款、沖帳及逾期帳款追蹤。
7. 保留款管理及釋放。
8. 驗收、結案及文件保存。

## 3. 作業依據

本程序依據下列文件與規範辦理：

- 公司內部控制制度。
- 銷售及收款循環相關管理辦法。
- 授權核決權限表。
- 合約管理辦法。
- 應收帳款管理辦法。
- 工程估驗與請款相關作業規範。
- 會計制度及相關帳務處理規定。
- 其他經公司核定之作業規章。

## 4. 名詞定義

| 名詞 | 說明 |
| --- | --- |
| 工程承攬案件 | 公司接受業主委託，依合約內容提供工程施作、管理或相關服務之案件。 |
| 合約 | 經雙方簽署或公司核准之承攬契約、訂單、報價確認書或具約束力之交易文件。 |
| 合約變更 | 合約金額、工期、施工範圍、付款條件或其他重要內容之追加、減少或調整。 |
| 估驗 | 依工程進度、合約條件及實際完成情形，向業主確認可請款金額之作業。 |
| 請款 | 依合約、估驗結果或付款條件向業主提出收款請求之作業。 |
| 保留款 | 依合約約定於請款或收款時保留，待驗收、保固或其他條件成就後收回之款項。 |
| 驗收 | 工程完成後，由業主、監造或相關權責單位確認工程內容符合合約要求之程序。 |

## 5. 權責單位

| 單位 | 主要權責 |
| --- | --- |
| 業務單位 | 負責客戶接洽、報價、合約洽談、合約送審、請款資料協調及逾期款項催收。 |
| 工程單位 | 負責工程進度管理、施工資料提供、估驗資料編製、驗收資料整理及工程變更確認。 |
| 財會單位 | 負責請款資料審核、發票開立、應收帳款入帳、收款沖帳及帳齡追蹤。 |
| 法務或合約審查單位 | 負責重大合約條款、付款條件、保固、違約責任及法律風險審查。 |
| 權責主管 | 依核決權限核准合約、變更、估驗、請款、折讓、呆帳及其他例外事項。 |
| 稽核室 | 依年度稽核計畫或專案需求，查核本程序之遵循情形與內控制度有效性。 |

## 6. 作業流程

傳統 SOP 作業流程如下：

```text
案件建立
  -> 合約審查與簽訂
  -> 合約變更或追加減
  -> 工程進度確認
  -> 估驗申請與審核
  -> 請款及發票開立
  -> 收款與沖帳
  -> 保留款管理
  -> 驗收與結案
  -> 文件歸檔
```

## 7. 作業程序

### 7.1 案件建立

1. 業務單位取得客戶需求或案件資料後，應建立案件基本資料。
2. 案件資料應包含客戶名稱、工程名稱、工程地點、預估金額、預估工期、付款條件及負責人員。
3. 新客戶或重大案件應依公司規定進行客戶資料審查或信用評估。
4. 案件建立後，應由權責主管審核案件承接之合理性。

### 7.2 合約審查與簽訂

1. 業務單位應依報價、議價結果及業主需求編製合約草案或取得業主合約版本。
2. 合約內容應至少包含工程範圍、工程金額、付款條件、工期、驗收方式、保固、保留款、違約責任及爭議處理條款。
3. 重大合約或特殊條款應送法務或合約審查單位審閱。
4. 合約簽訂前，應依公司授權核決權限辦理核准。
5. 合約完成簽署後，業務單位應將正本或電子檔送交指定單位保管，並通知工程及財會單位辦理後續作業。

### 7.3 合約變更及追加減

1. 工程執行期間，如發生施工範圍、金額、工期、付款條件或其他重要條件變動，應辦理合約變更或追加減作業。
2. 合約變更應載明變更原因、變更內容、金額影響、工期影響及相關附件。
3. 合約變更應經業務、工程、財會及權責主管審核後始得執行。
4. 未經核准之變更，不得作為估驗、請款或收入認列之依據。
5. 合約變更完成後，相關文件應併同原合約歸檔保存。

### 7.4 工程進度確認

1. 工程單位應依施工進度、工程日報、現場紀錄或業主確認資料，定期確認工程完成情形。
2. 工程進度資料應能支持後續估驗及請款作業。
3. 如工程進度與合約約定有重大差異，工程單位應通知業務單位及權責主管處理。
4. 涉及追加、減少或爭議項目者，應依合約變更程序辦理。

### 7.5 估驗作業

1. 工程單位應依合約約定、工程進度及可請款條件編製估驗資料。
2. 估驗資料應包含估驗期間、估驗項目、數量、金額、累計估驗金額及相關佐證。
3. 業務單位應協助確認業主估驗要求、請款時程及附件格式。
4. 估驗資料應送權責主管審核，必要時送財會單位檢核金額及合約條件。
5. 經核准之估驗資料，得作為後續請款依據。

### 7.6 請款及發票開立

1. 業務單位或財會單位應依已核准之合約、估驗資料或付款條件提出請款。
2. 請款資料應包含合約編號、請款期別、請款金額、稅額、扣款、保留款及應收金額。
3. 財會單位應檢核請款金額與合約、估驗資料及付款條件是否相符。
4. 發票開立應依稅法及公司會計制度辦理。
5. 請款及發票資料完成後，應通知業務單位追蹤業主付款情形。

### 7.7 應收帳款及收款

1. 財會單位應依請款及發票資料入帳應收帳款。
2. 收到款項時，財會單位應依銀行入帳資料、收款通知或相關憑證辦理沖帳。
3. 收款金額與應收帳款不符者，應查明原因並通知業務單位處理。
4. 業務單位應定期追蹤未收款項，必要時提出催收紀錄。
5. 逾期帳款應依公司規定提報權責主管，並評估後續處理方式。

### 7.8 保留款管理

1. 財會單位應依合約條件記錄保留款金額及到期條件。
2. 業務單位及工程單位應追蹤保留款可請領時點。
3. 保留款請領應檢附驗收、保固或合約約定之必要文件。
4. 保留款收回後，財會單位應辦理沖帳並更新相關紀錄。

### 7.9 驗收及結案

1. 工程完成後，工程單位應依合約及業主要求辦理驗收。
2. 驗收資料應包含完工資料、驗收紀錄、缺失改善紀錄及業主確認文件。
3. 如有驗收缺失，工程單位應追蹤改善並留存紀錄。
4. 案件結案前，應確認合約、變更、請款、收款、保留款及驗收文件是否完整。
5. 結案資料應依公司文件保存規定歸檔。

## 8. 控制重點

| 控制項目 | 控制說明 | 負責單位 |
| --- | --- | --- |
| 合約核准 | 合約簽訂前應依核決權限核准，重大條款應送相關單位審查。 | 業務 / 法務 / 權責主管 |
| 合約變更 | 合約變更或追加減應經正式核准，未核准變更不得請款。 | 業務 / 工程 / 權責主管 |
| 估驗依據 | 估驗應依合約、工程進度及相關佐證辦理。 | 工程 / 業務 |
| 請款審核 | 請款金額應與合約、估驗資料及付款條件相符。 | 財會 / 業務 |
| 收款沖帳 | 收款應依銀行入帳資料或收款憑證辦理沖帳。 | 財會 |
| 逾期追蹤 | 逾期帳款應定期追蹤並提報權責主管。 | 業務 / 財會 |
| 保留款 | 保留款應依合約條件記錄、追蹤及請領。 | 財會 / 業務 |
| 文件留存 | 合約、變更、估驗、請款、收款、驗收及結案資料應妥善保存。 | 各權責單位 |

## 10. 文件保存

1. 合約、合約變更、估驗、請款、收款、保留款、驗收及結案資料，應依公司文件保存規定保存。
2. 紙本文件應由權責單位妥善保管；電子文件應存放於公司指定系統或資料夾。
3. 文件保存期間應依公司規定、法令要求及稽核需求辦理。
4. 文件調閱應依公司權限規範辦理。

## 11. 異常處理

下列情形應由承辦單位查明原因，必要時提報權責主管：

- 合約尚未完成核准即開始執行。
- 合約變更未完成核准即估驗或請款。
- 請款金額與合約或估驗資料不符。
- 收款金額與應收帳款不符。
- 逾期帳款未依規定追蹤。
- 保留款逾期未請領或未收回。
- 驗收文件不完整。
- 重要文件未依規定歸檔。

異常事項應留存處理紀錄，包括發生原因、處理方式、責任單位、處理結果及核准紀錄。

## 12. 內部稽核

稽核室得依年度稽核計畫或專案需求，查核本程序執行情形。查核重點包括：

- 合約是否依規定審查及核准。
- 合約變更是否依程序辦理。
- 估驗及請款是否有適當依據。
- 應收帳款及收款沖帳是否正確。
- 逾期帳款是否定期追蹤。
- 保留款是否完整記錄及請領。
- 文件是否依規定保存。

稽核發現缺失時，應請權責單位提出改善措施並追蹤改善情形。

## 13. 修訂與施行

1. 本程序之訂定、修訂及廢止，應依公司規定辦理。
2. 本程序經權責主管核准後施行，修訂時亦同。
3. 各單位應依本程序辦理相關作業，並配合稽核查核及改善。

## 14. 與後續版本之對照定位

本 S 版代表傳統上市櫃 SOP / 內控制度 reality。

後續對照關係如下：

| 版本 | 定位 | 主要差異 |
| --- | --- | --- |
| S 版 | 傳統上市櫃 SOP / 內控制度 reality | 以流程、權責、表單、核准及文件保存為主。 |
| A 版 | TAAT Governance 進化版 | 強化 Workflow、KCP、文件治理、可維運性及跨部門治理語言。 |
| B 版 | TAAT Computer Audit / AI-assisted 深化版 | 強化系統資料、Audit Trail、Log、Cross-check、Suspicious Timeline 與例外清單。 |

S 版的價值在於提供市場現況 baseline。只有先看懂一般公司現在的 SOP 長什麼樣子，後續才容易說明 A 版與 B 版到底進化在哪裡。
