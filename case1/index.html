Documents

S / A / B 三份對照文件

S 版代表市場主流傳統 SOP 樣貌；A 版負責治理進化；B 版負責 Computer Audit 深度檢查。

S版：市場現況 一般上市櫃公司熟悉的 SOP / 內控制度樣貌，用來當 baseline。
A版：治理進化 在傳統 SOP 上加強 Workflow、KCP、文件治理與可維護性。

Boundary

這個案例不是什麼

  • 不是法律判定。
  • 不是系統自動認定違建。
  • 不是自動核准或否決工程款。
  • 不是對外服務提案。
  • 最終判斷仍由稽核主管、法務與專業人員負責。

Principles

本版文件原則

目前停在傳統治理文件與 Computer Audit 附加版本，不往更高階自動化延伸。

治理優先 先定義流程、權責、控制點與證據，再談系統檢查。
漸進導入 內控 / SOP → Computer Audit → Workflow / ERP → 系統留痕。
人工覆核 例外清單只是提示，稽核結論仍由人負責。
可追溯 每個流程、表單、附件與 Log 都要能回到同一條證據鏈。
